증권 API 연결 제안
작성 기준: 2026-05-28

결론:

- 가능하다.
- 다만 바로 자동주문부터 붙이지 않고, 조회 전용 → 모의/가상 주문 → 수동 승인 주문 → 제한적 자동주문 순서로 가야 한다.
- API 키와 계좌 비밀번호는 브라우저에 두지 않고 PC/스마트폰 로컬 서버의 환경변수 또는 암호화 저장소에만 둔다.


1. 화면 기본 방향

- 앱 첫 화면은 위클리 옵션 전략 탭으로 둔다.
- 지수 매매 탭은 옵션 진입의 근거 화면으로 둔다.
- 종목 신호 탭은 국내 주식 후보와 보유 종목 확인용으로 유지한다.


2. 연결 단계

1단계: 읽기 전용 연결
- 실시간 지수, 옵션 현재가, 호가, 미결제약정, 계좌 잔고만 조회한다.
- 주문 버튼은 비활성으로 둔다.

2단계: 주문 시뮬레이터
- 현재 알고리즘이 어떤 행사가, 프리미엄, 수량, 손절선을 선택했는지 주문 미리보기로 만든다.
- 실제 주문 대신 paper_order 로그만 남긴다.

3단계: 수동 승인 주문
- 앱이 매수/매도 후보를 만들면 사용자가 직접 승인 버튼을 눌러야 주문한다.
- 승인 전에는 종목, 행사가, 수량, 지정가, 예상 손실, 당일 누적손익을 한 화면에 보여준다.

4단계: 제한적 자동주문
- 하루 최대 손실 300만원, 12시 이후 40% 손절, 1계약 제한 같은 룰을 코드 레벨에서 강제한다.
- 자동주문은 처음부터 켜지 않고, 충분한 로그 검증 후 별도 스위치로만 허용한다.


3. 필요한 내부 모듈

broker_adapter
- 증권사별 API 차이를 감추는 공통 인터페이스
- get_quote, get_orderbook, get_positions, place_order, cancel_order, get_fills

strategy_engine
- 현재 JSON 전략을 읽어 신호와 주문 후보를 만든다.
- 자동으로 주문하지 않고 order_intent 객체만 생성한다.

risk_guard
- 당일 최대 손실, 계약수 제한, 시간대별 손절폭, 중복 주문, 미체결 시간 제한을 검사한다.
- risk_guard를 통과하지 못한 주문은 API로 보내지 않는다.

execution_gateway
- 실제 증권 API 호출 전 마지막 관문이다.
- 처음에는 paper 모드만 지원하고, 이후 live 모드를 추가한다.

audit_log
- 모든 신호, 주문 후보, 승인, 주문 응답, 체결, 취소, 오류를 파일 또는 DB에 남긴다.


4. 증권사 API 후보

한국투자증권 KIS Developers
- REST/WebSocket 기반으로 접근하기 좋다.
- 공식 포털에 국내주식, 국내선물옵션, 해외주식, 해외선물옵션 등의 API 문서 구분이 있다.
- 공식 샘플 코드와 테스트베드가 있어 프로토타입에 유리하다.
- 확인 URL: https://apiportal.koreainvestment.com/docs

키움 REST API
- 실전투자와 모의투자 App Key를 별도로 관리한다.
- OAuth 방식의 접근 토큰을 사용한다.
- IP 등록과 계좌 등록 절차가 필요하다.
- 확인 URL: https://openapi.kiwoom.com/intro/serviceInfo

LS증권 OPEN API
- 투자정보, 계좌정보, 매매주문 등 상품별 트레이딩을 제공한다고 안내되어 있다.
- API 서비스 항목에 국내파생 시세와 국내파생 계좌 주문이 분리되어 있어 옵션/파생 쪽 검토 후보로 좋다.
- 확인 URL: https://openapi.ls-sec.co.kr/intro


5. 안전장치 우선순위

- 기본값은 항상 paper 모드
- 실주문 모드는 별도 PIN 또는 로컬 패스워드 필요
- 1일 손실 한도 도달 시 주문 API 호출 자체 차단
- 동일 행사가 중복 주문 차단
- 미체결 주문 자동 취소/정정 기준 필요
- 실시간 잔고와 앱 내부 포지션이 다르면 주문 차단
- 장 마감/만기/호가 단절/네트워크 오류 시 주문 차단


6. 다음 개발 순서 제안

1. broker_api_plan.txt 기준으로 broker_adapter 인터페이스만 먼저 만든다.
2. 실제 증권사 연결 없이 paper broker를 붙여 주문 후보 로그를 쌓는다.
3. 위클리 옵션 화면에 주문 미리보기 패널을 추가한다.
4. 증권사를 하나 선택한 뒤 조회 API만 연결한다.
5. 모의투자 또는 테스트베드 주문을 붙인다.
6. 수동 승인 실주문을 제한적으로 붙인다.
